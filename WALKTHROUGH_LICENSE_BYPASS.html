1.3 Multi-Layer Protection (Original)

The original Rocket LMS uses a multi-layered defense approach:

  1. Layer 1 - Kernel: ionCube encrypted Kernel.php with license checks on boot
  2. Layer 2 - Middleware: LicenseCheck middleware validates on each request
  3. Layer 3 - Services: LicenseService classes verify with Envato API
  4. Layer 4 - Providers: Service providers register license event listeners
  5. Layer 5 - Models: PurchaseCode model stores and validates purchase codes
  6. Layer 6 - Routes: Encrypted routes redirect to license page if invalid

8. Remediation Recommendations

⚠️ For Software Vendor: This section provides recommendations to improve license protection.

8.1 Vulnerabilities Identified

ID Severity Description
VUL-001 HIGH License service classes are unencrypted PHP
VUL-002 HIGH Routes file can be replaced entirely
VUL-003 HIGH Middleware files are unencrypted
VUL-004 MEDIUM No runtime file integrity checks
VUL-005 MEDIUM Service providers easily neutralized
VUL-006 LOW License stored in plaintext database

8.2 Recommended Mitigations

P1 - Immediate:

  • Apply ionCube encryption to ALL license-related files (services, middleware, providers)
  • Implement file hash verification in encrypted Kernel.php

P2 - Short-term:

  • Move route definitions into encrypted controller constructors
  • Use unified error messages for license failures

P3 - Long-term:

  • Encrypt license data at rest with signed tokens
  • Implement periodic license revalidation with server

17. Limitations & Known Issues

  1. ionCube Encrypted Files: The core app/Http/Kernel.php contains ionCube encrypted license checks that cannot be fully disabled.
  2. External API Calls: Some features may still attempt API calls to verification servers (these will fail silently).
  3. Updates Incompatible: This bypass is specific to v2.1 and must be reapplied after any official updates.
